Projet Cloud — Azure IaC Windows Infrastructure as Code · Terraform

Déploiement d'une infrastructure Windows sur Microsoft Azure via Terraform : deux VMs Windows Server 2022, un poste client Windows 11, un réseau virtuel segmenté et des règles de sécurité NSG (RDP 3389 + SSH 22).

Microsoft Azure Terraform Windows Server 2022 Windows 11 VNet · NSG RDP · SSH
01 — Contexte du projet
Stage Paradigm — Versailles
Superviseur : Yasser Salimi · Janvier — Mars 2025

Dans le cadre du stage chez Paradigm, j'ai déployé une infrastructure cloud Windows complète sur Microsoft Azure en utilisant Terraform (Infrastructure as Code). L'objectif est de provisionner deux serveurs Windows Server 2022 (contrôleur de domaine et serveur applicatif) ainsi qu'un poste client Windows 11, reliés par un réseau virtuel Azure sécurisé par un NSG avec règles RDP et SSH.

Toute l'infrastructure est définie en code HCL (HashiCorp Configuration Language) et déployée en une seule commande terraform apply. Elle est reproductible, versionnée sur Git et supprimable proprement avec terraform destroy.

04 — NSG — Règles de sécurité RDP & SSH
Network Security Group — nsg-windows
RDP port 3389 · SSH port 22 · Deny all par défaut

Le NSG définit les règles de filtrage du trafic réseau. Deux règles inbound permettent l'accès à distance : RDP (3389) pour la connexion Bureau à distance Windows, et SSH (22) pour l'accès terminal sécurisé. Tout autre trafic entrant est bloqué par la règle implicite DenyAll.

Sécurité en production : Les règles RDP/SSH autorisent ici toutes les IPs sources (*) pour l'environnement de lab. En production, il faudrait restreindre à une IP spécifique ou à un plage Paradigm.

05 — Code Terraform
providers.tf + variables.tf
hashicorp/azurerm ~> 3.80 · Configuration de base
providers.tf
terraform {
  required_version = ">= 1.5.0"
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 3.80"
    }
  }
}

provider "azurerm" {
  features {}
  subscription_id = var.subscription_id
}
variables.tf
variable "subscription_id" { type = string }
variable "location"        { default = "westeurope" }
variable "rg_name"         { default = "rg-windows-infra" }
variable "admin_username"  { default = "adminparadigm" }
variable "admin_password"  { type = string; sensitive = true }
variable "tags" {
  default = {
    project    = "windows-infra"
    env        = "lab"
    managed_by = "terraform"
    stage      = "paradigm-2025"
  }
}

  name                 = "subnet-vms"

# ── IP Publiques (une par VM) ────────────────────────────────
resource "azurerm_public_ip" "dc" {

  sku                 = "Standard"
  tags                = var.tags
}
# (idem pour pip-win-app et pip-win-client)

# ── Network Interfaces ───────────────────────────────────────
resource "azurerm_network_interface" "dc" {

  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name

}

main.tf — NSG + Règles RDP & SSH
Network Security Group avec 3 règles inbound
main.tf — NSG + association subnet
# ── Network Security Group ───────────────────────────────────
resource "azurerm_network_security_group" "main" {
  name                = "nsg-windows"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name

  # ── Règle RDP — Bureau à distance Windows ─────────────────
  security_rule {
    name                       = "Allow-RDP"
    priority                   = 100
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "3389"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
    description                = "Acces Bureau a distance (RDP) vers les VMs Windows"
  }

  # ── Règle SSH — Terminal sécurisé ────────────────────────
  security_rule {
    name                       = "Allow-SSH"
    priority                   = 110
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "22"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
    description                = "Acces SSH pour administration distante"
  }

  # ── WinRM — Gestion Windows distante ────────────────────
  security_rule {
    name                       = "Allow-WinRM"
    priority                   = 200
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_ranges    = ["5985", "5986"]
    source_address_prefix      = "VirtualNetwork"
    destination_address_prefix = "*"
    description                = "WinRM interne au VNet uniquement"
  }

  tags = var.tags
}

# ── Association NSG ↔ Subnet ─────────────────────────────────
resource "azurerm_subnet_network_security_group_association" "main" {
  subnet_id                 = azurerm_subnet.vms.id
  network_security_group_id = azurerm_network_security_group.main.id
}

07 — Tests & Validation
Validation de l'infrastructure
Connexion RDP · SSH · Ping inter-VM · NSG effectif
1
Vérification Azure Portal → Resource Group
Navigation dans rg-windows-infra → 15 ressources créées. Les 3 VMs affichées en statut Running. NSG associé au subnet visible dans les détails.
2
Connexion RDP → vm-win-dc (Windows Server)
mstsc /v:<dc_public_ip>:3389 → Connexion Bureau à distance établie avec adminparadigm. Windows Server 2022 accessible.
3
Connexion RDP → vm-win-app et vm-win-client
Même procédure avec les IPs publiques des outputs Terraform. Les 3 VMs accessibles en RDP via les règles NSG Allow-RDP.
4
Test SSH → vm-win-dc (OpenSSH Server)
ssh adminparadigm@<dc_public_ip> → Connexion SSH réussie (OpenSSH Server préinstallé sur WinServer 2022). Règle Allow-SSH validée.
5
Ping inter-VMs (VNet interne)
Depuis vm-win-dc : ping 10.0.1.5 (vm-win-app) → Réponse reçue. Les 3 VMs communiquent via le subnet 10.0.1.0/24.
6
Vérification NSG — Règle DenyAll
Test d'un port non autorisé (ex: 8080) depuis l'extérieur → Connexion refusée (timeout). Le NSG bloque correctement tout trafic non explicitement autorisé.
