Projet Cloud — Azure IaC Azure Infrastructure as Code · Terraform

Déploiement automatisé d'une infrastructure cloud sécurisée sur Microsoft Azure via Terraform : Resource Group, Virtual Network, NSG (SSH / HTTP / HTTPS), VM Ubuntu Server avec installation automatique d'un service web (Nginx) via cloud-init.

Microsoft Azure Terraform Ubuntu Server 22.04 Nginx · Cloud-init VNet · NSG SSH · HTTP · HTTPS
01 — Contexte du projet
Stage Paradigm — Versailles
Superviseur : Yasser Salimi · Janvier — Mars 2025

Dans le cadre du stage chez Paradigm, j'ai déployé une infrastructure cloud complète sur Microsoft Azure en utilisant Terraform (Infrastructure as Code). L'objectif est de provisionner automatiquement une VM Ubuntu Server 22.04 avec son réseau virtuel sécurisé, son groupe de sécurité réseau (NSG) et un service web Nginx installé automatiquement au premier démarrage via cloud-init.

Toute l'infrastructure est définie en code HCL (HashiCorp Configuration Language) et déployée en une seule commande terraform apply. Elle est reproductible, versionnée sur Git et supprimable proprement avec terraform destroy.

03 — NSG — Règles SSH · HTTP · HTTPS
Network Security Group — nsg-paradigm
SSH 22 · HTTP 80 · HTTPS 443 · Deny all par défaut

Le NSG filtre le trafic réseau de la VM. Trois règles inbound sont définies : SSH (22) pour l'administration distante, HTTP (80) et HTTPS (443) pour l'exposition du service Nginx. Tout autre trafic entrant est bloqué par la règle implicite DenyAll.

Sécurité en production : La règle SSH autorise ici toutes les IPs sources (*) pour l'environnement de lab. En production, restreindre à l'IP de l'administrateur ou utiliser un Bastion Azure.

04 — Machine Virtuelle — Ubuntu + Nginx (cloud-init)
vm-paradigm-web — Ubuntu Server 22.04 LTS
Standard_B1s · Nginx installé automatiquement via cloud-init

La VM Ubuntu Server 22.04 est déployée avec une IP privée statique (10.0.1.4) et une IP publique. Le service Nginx est installé et démarré automatiquement au premier boot grâce à un script cloud-init passé via le paramètre custom_data de Terraform.

Nginx — Service Web
Installation automatique cloud-init

cloud-init est un standard de configuration des VMs Linux au premier démarrage. Terraform passe le script en base64 via custom_data ; Azure l'exécute automatiquement dès que la VM démarre.
cloud-init.yaml — Installation automatique Nginx
#cloud-config
package_update: true
package_upgrade: true

packages:
  - nginx
  - curl
  - git

runcmd:
  - systemctl enable nginx
  - systemctl start nginx
  - echo "<h1>Paradigm — Infrastructure Azure Terraform</h1>" > /var/www/html/index.html

final_message: "Nginx deployed by Terraform — Paradigm 2025"

05 — Code Terraform
providers.tf + variables.tf
hashicorp/azurerm ~> 3.80 · Configuration de base
providers.tf
terraform {
  required_version = ">= 1.5.0"
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 3.80"
    }
  }
}

provider "azurerm" {
  features {}
  subscription_id = var.subscription_id
}
variables.tf
variable "subscription_id" { type = string }
variable "location"        { default = "westeurope" }
variable "rg_name"         { default = "rg-paradigm-infra" }
variable "admin_username"  { default = "adminparadigm" }
variable "ssh_public_key"  { type = string; description = "Clé SSH publique (~/.ssh/id_rsa.pub)" }
variable "tags" {
  default = {
    project    = "paradigm-infra"
    env        = "lab"
    managed_by = "terraform"
    stage      = "paradigm-2025"
  }
}
main.tf — Réseau
Resource Group · VNet · Subnet · IP Publique · NIC · NSG
main.tf — Resource Group + VNet + Subnet + IP + NIC + NSG
# ── Resource Group ───────────────────────────────────────────
resource "azurerm_resource_group" "main" {
  name     = var.rg_name
  location = var.location
  tags     = var.tags
}

# ── Virtual Network ──────────────────────────────────────────
resource "azurerm_virtual_network" "main" {
  name                = "vnet-paradigm"
  address_space       = ["10.0.0.0/16"]
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name
  tags                = var.tags
}

# ── Subnet principal ─────────────────────────────────────────
resource "azurerm_subnet" "main" {
  name                 = "subnet-main"
  resource_group_name  = azurerm_resource_group.main.name
  virtual_network_name = azurerm_virtual_network.main.name
  address_prefixes     = ["10.0.1.0/24"]
}

# ── IP Publique statique ─────────────────────────────────────
resource "azurerm_public_ip" "vm" {
  name                = "pip-paradigm-vm"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name
  allocation_method   = "Static"
  sku                 = "Standard"
  tags                = var.tags
}

# ── Network Interface ────────────────────────────────────────
resource "azurerm_network_interface" "vm" {
  name                = "nic-paradigm-vm"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name

  ip_configuration {
    name                          = "ipconfig-vm"
    subnet_id                     = azurerm_subnet.main.id
    private_ip_address_allocation = "Static"
    private_ip_address            = "10.0.1.4"
    public_ip_address_id          = azurerm_public_ip.vm.id
  }
  tags = var.tags
}

# ── NSG — SSH + HTTP + HTTPS ─────────────────────────────────
resource "azurerm_network_security_group" "main" {
  name                = "nsg-paradigm"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name

  security_rule {
    name                       = "Allow-SSH"
    priority                   = 100
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "22"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }

  security_rule {
    name                       = "Allow-HTTP"
    priority                   = 110
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "80"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }

  security_rule {
    name                       = "Allow-HTTPS"
    priority                   = 120
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "443"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }

  tags = var.tags
}

# ── Association NSG ↔ Subnet ─────────────────────────────────
resource "azurerm_subnet_network_security_group_association" "main" {
  subnet_id                 = azurerm_subnet.main.id
  network_security_group_id = azurerm_network_security_group.main.id
}

main.tf — VM Ubuntu Server 22.04 + outputs
# ── VM Ubuntu Server 22.04 LTS ───────────────────────────────
resource "azurerm_linux_virtual_machine" "web" {
  name                = "vm-paradigm-web"
  resource_group_name = azurerm_resource_group.main.name
  location            = azurerm_resource_group.main.location
  size                = "Standard_B1s"
  admin_username      = var.admin_username
  network_interface_ids = [azurerm_network_interface.vm.id]

  # ── Authentification SSH par clé ─────────────────────────
  admin_ssh_key {
    username   = var.admin_username
    public_key = var.ssh_public_key
  }

  os_disk {
    caching              = "ReadWrite"
    storage_account_type = "Standard_LRS"
    disk_size_gb         = 30
  }

  source_image_reference {
    publisher = "Canonical"
    offer     = "0001-com-ubuntu-server-jammy"
    sku       = "22_04-lts-gen2"
    version   = "latest"
  }

  # ── cloud-init : installe Nginx au premier boot ──────────
  custom_data = base64encode(file("cloud-init.yaml"))

  tags = var.tags
}

# ── outputs.tf ────────────────────────────────────────────────
output "vm_public_ip" {
  value       = azurerm_public_ip.vm.ip_address
  description = "IP publique de la VM — accès SSH et Nginx"
}

output "nginx_url" {
  value       = "http://${azurerm_public_ip.vm.ip_address}"
  description = "URL d'accès au service Nginx"
}

06 — Déploiement Terraform
Workflow de déploiement
init → plan → apply → verify → destroy
Terminal — Déploiement complet
# ── 1. Connexion Azure ───────────────────────────────────────
$ az login
# Si MFA bloqué : az login --use-device-code

# ── 2. Variables sensibles ───────────────────────────────────
$ export TF_VAR_subscription_id="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
$ export TF_VAR_ssh_public_key="$(cat ~/.ssh/id_rsa.pub)"

# ── 3. Initialisation des providers ──────────────────────────
$ terraform init
Terraform has been successfully initialized!

# ── 4. Plan — Aperçu des ressources à créer ──────────────────
$ terraform plan
Plan: 8 to add, 0 to change, 0 to destroy.
# RG + VNet + Subnet + PIP + NIC + NSG + NSG assoc + VM

# ── 5. Déploiement effectif ───────────────────────────────────
$ terraform apply -auto-approve
azurerm_resource_group.main: Creating...
azurerm_virtual_network.main: Creating...
azurerm_network_security_group.main: Creating...
azurerm_linux_virtual_machine.web: Creating...  [~3 min]
Apply complete! Resources: 8 added, 0 changed, 0 destroyed.

Outputs:
vm_public_ip = "52.174.x.x"
nginx_url    = "http://52.174.x.x"

# ── 6. Nettoyage (fin de lab) ─────────────────────────────────
$ terraform destroy -auto-approve
Destroy complete! Resources: 8 destroyed.

07 — Tests & Validation
Validation de l'infrastructure
SSH · Nginx HTTP · NSG effectif · cloud-init confirmé
1
Vérification Azure Portal → Resource Group
Navigation dans rg-paradigm-infra → 8 ressources créées. VM affichée en statut Running. NSG associé au subnet visible dans les détails.
2
Connexion SSH → vm-paradigm-web
ssh adminparadigm@<vm_public_ip> → Connexion SSH établie par clé RSA. Ubuntu Server 22.04 accessible. Règle Allow-SSH du NSG validée.
3
Vérification Nginx — service actif
systemctl status nginx → Active (running). cloud-init a bien installé et démarré Nginx au premier boot.
4
Test HTTP → accès via navigateur
http://<vm_public_ip> dans le navigateur → Page Nginx accessible. Règle Allow-HTTP (port 80) du NSG validée.
5
Vérification NSG — Règle DenyAll effective
Test d'un port non autorisé (ex: 8080) depuis l'extérieur → Connexion refusée (timeout). Le NSG bloque correctement tout trafic non explicitement autorisé.
6
Terraform destroy — Nettoyage propre
terraform destroy -auto-approve → Les 8 ressources supprimées proprement. Le Resource Group disparaît du portail Azure. Infrastructure reproductible.
