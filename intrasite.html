Configuration CLI IOS — RTW-Chasseneuil
3 interfaces + description + agent relais DHCP

Configuration des trois interfaces du routeur par zone réseau. L'interface LAN Bureaux reçoit en plus la commande ip helper-address pour relayer les requêtes DHCP vers SRVINTRA (10.2.0.1).

RTW-Chasseneuil — configuration complète
! ── Nom du routeur ─────────────────────────────────
Router(config)# hostname RTW-Chasseneuil

! ── LAN Bureaux (192.168.2.0/24) ──────────────────
RTW(config)# interface FastEthernet0/2
RTW(config-if)# description LAN_Bureaux_BatA_BatB
RTW(config-if)# ip address 192.168.2.254 255.255.255.0
RTW(config-if)# ip helper-address 10.2.0.1
RTW(config-if)# no shutdown
RTW(config-if)# exit

! ── LAN Serveurs (10.2.0.0/24) ────────────────────
RTW(config)# interface FastEthernet0/0
RTW(config-if)# description LAN_Serveurs_Internes
RTW(config-if)# ip address 10.2.0.254 255.255.255.0
RTW(config-if)# no shutdown
RTW(config-if)# exit

! ── Zone DMZ (172.16.2.0/24) ──────────────────────
RTW(config)# interface FastEthernet0/1
RTW(config-if)# description Zone_DMZ_Serveurs_Publics
RTW(config-if)# ip address 172.16.2.254 255.255.255.0
RTW(config-if)# no shutdown
RTW(config-if)# exit

! ── Sauvegarde & vérifications ────────────────────
RTW# copy running-config startup-config
RTW# show ip interface brief
RTW# show ip route

Configuration VyOS — Routage & DHCP Relay
Tâche 2A — Interfaces + relais vers Windows Server

VyOS remplace le routeur Cisco en environnement virtuel. Il assure le routage entre LAN Bureaux et LAN Serveurs, et relaie les requêtes DHCP vers Windows Server 2022.

VyOS — Interfaces + routage + DHCP relay
# ── Mode configuration ──────────────────────────────
vyos@vyos:~$ configure

# ── Interface LAN Bureaux ────────────────────────────
vyos# set interfaces ethernet eth0 address 192.168.2.254/24
vyos# set interfaces ethernet eth0 description LAN-Bureaux

# ── Interface LAN Serveurs (Windows Server) ──────────
vyos# set interfaces ethernet eth1 address 10.2.0.254/24
vyos# set interfaces ethernet eth1 description LAN-Serveurs

# ── DHCP relay vers Windows Server ──────────────────
vyos# set service dhcp-relay interface eth0
vyos# set service dhcp-relay server 10.2.0.1
vyos# set service dhcp-relay relay-options relay-agents-packets replace

# ── Appliquer et sauvegarder ─────────────────────────
vyos# commit
vyos# save

# ── Vérifications ────────────────────────────────────
vyos@vyos:~$ show interfaces
vyos@vyos:~$ show ip route
vyos@vyos:~$ show service dhcp-relay

Tâche 2B — AD DS · DNS · DHCP · OUs · Utilisateurs · Groupes

Installation des rôles AD DS, DNS et DHCP sur Windows Server 2022 puis promotion en contrôleur de domaine. Création de la structure organisationnelle et des comptes selon le schéma TiersLieux86.

Domaine : chasseneuil.tierslieux86.fr · NetBIOS : CHASSENEUIL · IP fixe Windows Server : 10.2.0.1 · DNS : 127.0.0.1 après promotion
PowerShell (Admin) — Rôles + Promotion DC
# ── 1. Installer AD DS + DHCP ────────────────────────
PS C:\> Install-WindowsFeature -Name AD-Domain-Services,DHCP -IncludeManagementTools

# ── 2. Promouvoir en contrôleur de domaine ───────────
PS C:\> Install-ADDSForest `
    -DomainName "chasseneuil.tierslieux86.fr" `
    -DomainNetbiosName "CHASSENEUIL" `
    -InstallDns:$true `
    -SafeModeAdministratorPassword (ConvertTo-SecureString "P@ssw0rd!" -AsPlainText -Force) `
    -Force:$true
# → Le serveur redémarre automatiquement

# ── 3. Créer les Unités d'Organisation ──────────────
PS C:\> New-ADOrganizationalUnit -Name "Administration" -Path "DC=chasseneuil,DC=tierslieux86,DC=fr"
PS C:\> New-ADOrganizationalUnit -Name "Adherents"     -Path "DC=chasseneuil,DC=tierslieux86,DC=fr"
PS C:\> New-ADOrganizationalUnit -Name "Esporting"    -Path "DC=chasseneuil,DC=tierslieux86,DC=fr"

# ── 4. Créer les groupes de sécurité ─────────────────
PS C:\> New-ADGroup -Name "GRP_Employes"     -GroupScope Global -Path "OU=Administration,DC=chasseneuil,DC=tierslieux86,DC=fr"
PS C:\> New-ADGroup -Name "GRP_Responsables" -GroupScope Global -Path "OU=Administration,DC=chasseneuil,DC=tierslieux86,DC=fr"
PS C:\> New-ADGroup -Name "GRP_Esporting"   -GroupScope Global -Path "OU=Esporting,DC=chasseneuil,DC=tierslieux86,DC=fr"

# ── 5. Créer un utilisateur employé (exemple) ────────
PS C:\> New-ADUser `
    -Name "Jean Dupont" -SamAccountName "jdupont" `
    -UserPrincipalName "jdupont@chasseneuil.tierslieux86.fr" `
    -Path "OU=Administration,DC=chasseneuil,DC=tierslieux86,DC=fr" `
    -AccountPassword (ConvertTo-SecureString "P@ssw0rd!" -AsPlainText -Force) `
    -Enabled $true
PS C:\> Add-ADGroupMember -Identity "GRP_Employes" -Members "jdupont"

Dossiers partagés & Droits NTFS
Tâche 2C — Lecture (employés) / Écriture (responsables)

Partage créé sur le DC avec droits différenciés par groupes AD. Employés = lecture seule, Responsables = lecture + écriture. Les deux niveaux (SMB + NTFS) sont configurés.

Windows applique le droit le plus restrictif entre SMB et NTFS. Configurer les deux niveaux est obligatoire pour les droits effectifs corrects.
PowerShell (Admin) — Partage + droits NTFS
# ── Créer le dossier ────────────────────────────────
PS C:\> New-Item -Path "C:\Partages\TiersLieux86" -ItemType Directory

# ── Créer le partage SMB avec droits par groupe ──────
PS C:\> New-SmbShare `
    -Name "TiersLieux86" `
    -Path "C:\Partages\TiersLieux86" `
    -FullAccess   "Administrateurs" `
    -ChangeAccess "GRP_Responsables" `
    -ReadAccess   "GRP_Employes"

# ── Droits NTFS ──────────────────────────────────────
PS C:\> $acl = Get-Acl "C:\Partages\TiersLieux86"
PS C:\> $r1 = New-Object System.Security.AccessControl.FileSystemAccessRule("GRP_Employes","ReadAndExecute","ContainerInherit,ObjectInherit","None","Allow")
PS C:\> $r2 = New-Object System.Security.AccessControl.FileSystemAccessRule("GRP_Responsables","Modify","ContainerInherit,ObjectInherit","None","Allow")
PS C:\> $acl.SetAccessRule($r1) ; $acl.SetAccessRule($r2)
PS C:\> Set-Acl "C:\Partages\TiersLieux86" $acl

# ── Vérification ─────────────────────────────────────
PS C:\> Get-SmbShareAccess "TiersLieux86"
PS C:\> Get-Acl "C:\Partages\TiersLieux86" | Format-List
Groupe ADPermission SMBPermission NTFSAccès effectif
GRP_EmployesReadReadAndExecuteLecture seule
GRP_ResponsablesChangeModifyLecture + Écriture
AdministrateursFull ControlFull ControlContrôle total
